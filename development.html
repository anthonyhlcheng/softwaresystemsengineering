Software Development

"Software development is the process of computer programming, documenting, testing, and bug fixing involved in creating and maintaining applications and frameworks resulting in a software product."1

Overview History Details Models Future Research Sources

Overview

Software development invovles writing and maintaining code such that it meets requirements that a client or a firm sets; meets a need from consumers or for personal uses.

Like any other project, there are steps to take when developing software. The Systems Development Life Cycle helps guide developers through the process to give the project more structure and thus eventually lead to a very successful product.

History

On 29 June 1956, Herbert D. Benington described a similar pattern to the Waterfall Model, and this is the first time that the concept is brought up.

In 1983, the paper was republished by Benington. Also, he pointed out that the process depends on a prototype.

In 1970, an article by Winston W. Royce gave the first formal description of the Waterfall model, although the term 'Waterfall Model' wasn't used in that article.

The earliest use of the term "Waterfall" may have been in a 1976 paper by Bell and Thayer.

Details

Analysis stage

GANTT chart4 -- the chart that used to manage tasks and create the timetable. It looks like some long bars place on a time schedule, showing the starting time of a task and the total amount time need for this task. The unit could be days, weeks or months.

Implementation and testing

White box testing5 -- The testing used by the programmers or developers of the program. They will run the written code line by line and see if codes are doing the things as people expected. The testing will be used when a module is finished and after all modules are combined together. For example, a project has three parts named A, B, and C . Once the part A is developed, the programming will test it using white box testing and so as part B and part C. Then they will combine all A, B, and C and test the whole program.

Black Box testing6 -- The testing used by developers and the client of this project. It's used after the whole program pass through the white box testing. In this testing, people will run this program without view the code and see if the program reach all the requirement that the client has. Typically they will give numerous of input to the program and see if it can return all the expected outputs. The client has the right to decide if he or she are going to accept this program or application. Once the client satisfies with it, the implementation stage is ended.

Models

Waterfall Model

One of the most basic concepts based off the Development Life Cycle process as shown in the image above. It is straight-forward to understand given the sequential design process which is why up to third of businesses in the world still consider using this model.2 However, if anything goes wrong at any of the later stages, then it is expected that the project must be restarted to an acceptable earlier phase.3

Agile Model

The combination of lots of development life cycles and each cycle is used to develop a part of the whole project or a single module. One of the advantages of this model is that if there are some checks on the project then there is less of a chance of project failure. It can provide some results, while you might get nothing while using some models like the waterfall.

The Future

Specific methodologies are inefficient in some particular way - agile for instance can lead to inefficiencies within large organisations. It has also been noted that many methodologies are old and incompatible with modern development standards. Researchers have been trying to create one that has a one size fits all approach - the most optimal methodology with the least trade-offs. With the current problems with some methodologies, companies have already started experimenting by making hybrids to try and extract the best characteristics and form a new model.

Research

V-Model7 -- is also known as Verification and Validation model, extension on waterfall model. In this model, every single phase there is a directly associated test phase, including testing like acceptance testing and integration testing.

SSADM8 (Structured systems analysis and design method) -- a specific version of waterfall. It’s for the analysis and design of infomation systems. Three most important techques used are Logical Data Modeling, Data Flow Modeling, Entity Event Modeling.
