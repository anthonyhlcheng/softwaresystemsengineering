Software Deployment

"Software Deployment is all of the activities that make a software system available for use."1

Overview Processes Compatibility Release Installation Version Uninstall Research Sources

Overview

Software deployment is the term given to a broad set of processes which make software available to use. It involves preparing the software for users so that they could install the software and use it. It is normally the last stage of software development and it consists of a number of processes.

Processes

The following is a list of all the processes involved in software deployment:

Compatibility Testing
Release
Installation
Version Tracking
Update
Uninstall

Compatibility Testing

This process allows the developer to test if the software runs as expected in different environments. For example, a developer may want to check that his software executes appropriately on different operating systems or a web application running on a different browser or mobile device. This type of testing ensures the user will be able to use the program effectively regardless of the environment they choose to use.

Release

This process is done after development has been completed. The process involves transferring the software package to the point of sale (e.g website, disc).

Installation

This deployment process is normally done by the user. The user would often download an installation file from the point of sale and then follow on-screen instructions in order to install the software. The installation files are normally specific to different environments. The download site would normally allow the user to choose the operating system and whether it is a 32-bit or 64-bit OS. This ensures that the user could use the software within their environment. The Windows operating system usually comes with software installation files in the .EXE format whereas an OS X machine would normally require a .pkg file. These installation files only successfully run in their specific environment.

Version Tracking and Software Update

Version tracking keeps track of the latest software version and often notifies the user if there is a new update available. Software updates could bring improvements, bug-fixes and security features to a program. Updates could be installed manually by users or automatically.

Installation

Software packages usually come with a 'wizard' that allows the user to uninstall the program if the user wishes to do so for any reason. Sometimes, the uninstall wizard also links to a website asking the user why they chose to uninstall the program.

Research

As Software deployment is a set of processes that are always done after software is developed. As the processes involved in software development are fairly straightforward and effective, there is no significant research currently happening specifically with regards to the underlying processes that occur in software deployment. Over time in the past, we have seen some significant improvements in the way software deployment occurs. Examples of this include but are not limited to: automatic updates, availability of software at trusted online marketplaces, websites enhanced so that they can effectively be used on mobile devices or tablets. Small but highly effective changes like these will continue to make our lives easier.
