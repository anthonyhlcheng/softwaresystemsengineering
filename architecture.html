Software Architecture

Software Architecture refers to the relationships between software elements.

Overview Architecture Types Significance Research

Overview

Typically software has to do three things: retrieve and store data, perform calculations using that data and finally display the data. How a software system decides to make these three elements work together is what software architecture defines. A prominent type of software architecture is the MVC architecture, where a Model takes care of retrieving and storing data and performing calculations on the data, a View takes care of displaying and modifying the data and a Controller acts as the link between the Model and the View by telling the View to update if the Model changes and vice versa.

Software architecture typically only focus on the elements that are hard to change once the system is developed. For instance the software architecture is not concerned with how you decide to show data in the View, but it will define how you send data from the Model to the Controller. Making these simple rules early in the development process can save loads of time later.

Significance

During the software development process a system will be changed many times, it is very uncommon to write a program in one go and expect it to work perfectly, but some parts of the system are much easier to change than others. Think about the MVC architecture again; changing the way that the data is displayed is quite simple: just change the code in the View part of the software, however, if you want to start changing the way that data is passed around in the system, for example, if you want some data to be loaded in the Controller instead of the Model, then you suddenly have to change the entire architecture. This change will be time consuming and costly, and this is why software architecture is important.

Research

Software Architecture is a consideration in many research projects as it is a fundamental part of most software systems. For example, recent research from DAASE, which UCL CS is a member of, found an algorithm to automatically optimize the development process at the architectural level. Another research project from UCL has found a method of developing software architecture models from requirements models. Thereby generating architectural rules based given a set of system requirements.
