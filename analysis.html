Software Analysis

Software Analysis is a way of checking how good your software application is, in terms of how well it performs the tasks it’s supposed to perform; how robust it is and how fast it does those tasks.

Overview Tools More Details Research People References

Overview

Software Analysis is done by extracting the models of software systems and these models are simulated. This can provide insight into some specific software systems and provide data to create general insights. An example of this may improving a web searching application by analysing the algorithm that it uses and possibly changing it to a better one. There are research being done to create software that can automate the process of software analysis.

There are two types of software analysis:
• Instrumentation
It involves placing markers in the code to monitor the level of the code’s performance. This is done by using instructions to log activity of specific parts of the system. This is useful to analyse areas with potential problems while leaving the checked areas. It does have limitations, it can only be used if the code that is to be checked is actually executed. If the program never reaches that point, then instrumentation at that point will not collect any data. Some types of instrumentation can cause the execution time to increase.

• Code Sampling
This is done by inspecting the execution of code at fixed intervals. This is less accurate then Instrumentation but gives a wider coverage of the code. Since the method is less accurate, there is more of a chance for errors to be missed.

Tools

There are various types of software analysis tools. They are Code Coverage; Instruction Trace; Memory analysis and Performance analysis.

• Code coverage measures how much code has been executed
• Instruction trace creates a log of exactly what happens as the code runs.
• Memory analysis looks at the memory usage of the code and finds errors.
• Performance analysis finds performance bottle necks to improve execution speed.

More Details

This is similar to a method of testing software called debugging. Debugging is used to find a known problem in the code. It also involves tracing the code execution and monitoring the variables.

The main difference between software analysis and debugging is that software analysis can be carried out while the program is running and not have to pause. This is especially important if the software is an embedded one where the system cannot always be halted.

The advantages of using software analysis are: accelerating the development of software, improving the software quality and creating software that gives maximum performance. By analysing software, developers can write software in a shorter time. More efficient software can be created, which is heavily used in real time systems that require it.

Research

There is currently research being cared out in the Centre for Research and Testing (CREST) into new methods of software analysis. One method they found is genetic improvement. It involves treating the source code of a program as genetic material. This means that small edits are made to the source code and then algorithms are used to find versions of the code that are improved from the original. This allows programs to be upgraded without losing core functionality. An example of this is Java byte code produced from existing human written programs being improved using genetic improvement.
Genetic improvement can be used to target areas of mobile apps which can be improved such as energy and bandwidth usage. It can also create multiple versions of a program that are tailored to a particular use. Which can make producing bespoke software much easier.
